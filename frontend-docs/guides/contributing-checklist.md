# Contributing Checklist

## Before Coding
- Confirm feature/domain ownership under `src/`.
- Identify affected tests and fixtures.
- Check API contract changes with backend schema.

## During Coding
- Keep changes scoped to relevant feature directory when possible.
- Update domain and repository mappings together.
- Preserve typing discipline and existing architectural boundaries.

## Before Merging
- Run lint and tests.
- Validate key flows manually for changed features.
- Update docs in `docs/features/` and `docs/reference/` as needed.

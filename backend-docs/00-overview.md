# Backend Docs Overview

## Purpose

This is the entrypoint for backend documentation. It explains what the backend docs are for, how they are organized, and where to start for each role.

## Goals

- Keep documentation compact and navigable.
- Explain system intent and contracts before low-level detail.
- Centralize policy and avoid duplicate guidance.

## Audience

- New contributors: understand architecture and workflow quickly.
- Maintainers: operate and evolve services safely.
- Integrators: rely on stable API behavior and compatibility policy.

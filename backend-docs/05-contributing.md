# Backend Contributing

## Purpose

Define contributor workflow, review standards, and documentation responsibilities.

## Contribution Expectations

- Prefer small, focused PRs.
- Update tests and docs alongside behavior changes.
- Keep contract and lifecycle docs aligned with code changes.

## Review Checklist

- Architecture impact considered.
- API contract impact documented.
- Compatibility/deprecation impact documented.
- Operational impact documented when relevant.

## Documentation Rule

No backend behavior change is complete until affected core docs are updated.
